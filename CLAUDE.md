## Claude Code 固有の指示

### 1. 作業着手前宣言

作業を始める前に以下を口頭で宣言する:

> 「最優先5原則を確認しました。作業対象: {ディレクトリ}、フェーズ: {N}、チケット: {ID}」

最優先5原則は `/AGENTS.md` 冒頭の blockquote を参照。

### 2. 設計書編集時の実装状況更新

設計書を編集する場合、実装状況セクションを必ず最新の Step / フェーズ状態に同期させる。実装と設計書更新は原則として別コミット (詳細は `/AGENTS.md` §5)。

### 3. チケット単位の作業

複数チケットで構成された作業は、チケット単位で commit を分け、`chore(<phase>): <内容> (Ticket {ID})` 形式のコミットメッセージを使う。チケット完了時に進捗報告フォーマット (KICKOFF の §7 等で定義) を出力する。

### 4. 不明点の扱い

設計判断・チケット範囲・既存実装に不明点がある場合、勝手に決めず必ずユーザーに確認する (最優先5原則 §1)。
